The FRL interpreter

The author writes what. The interpreter decides how.

A check's property namespace is the signal that routes its evaluation — the rule never names a mechanism.

stable · demoed

MCP path

Top-level properties, PERMISSIONS, LINEAGE — answered from Fabric REST metadata via the Fabric MCP proxy. No Spark.

experimental

Notebook path

delta · schema · access — need Spark on the data plane, so the interpreter runs a Fabric notebook. Designed in the language; runtime not yet stable.

The same principle makes FRL forward-compatible: new properties and item types route automatically, no language change required.
